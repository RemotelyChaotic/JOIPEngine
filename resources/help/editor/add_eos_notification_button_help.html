Add Notification Create and Close Instructions

This button adds a notification create or close instruction to the currently opened eos script in the EOS Editor.


The following parameters can be changed for notification create instructions:

  • id (json attr: id): The id of the notification. Notifications can be referenced from Eval Instructions via the id and all notifications will be sorted according to the id from top to bottom.
  • title (json attr: title): This is the static text on a notification. Can be html and can contain evaluator expressions: <eval>//code here</eval>.
  • button (json attr: buttonLabel): The button label of the notification. A button is only shown if the label string is not empty. Can be html and can contain evaluator expressions: <eval>//code here</eval>.
  • timerDuration (json attr: startAt): The duration for which the notification is shown. If left empty the notification will be shown indefinitely. Needs to follow the format <weeks>w<days>d<hours>h<minutes>m<seconds>s<milis>ms or be a plain number denoting miliseconds. For example: 2m30s or 100.


The following parameters can be changed for notification close instructions:

  • id (json attr: id): The id of the notification to close. No timeout or button commands will be executed for a notification closed this way.


notification.create UInotification.remove UI

Both nodes in serialised JSON form:

{
  "notification.create": {
      "id": "myId",
      "title": "Notification",
      "buttonLabel": "Click Me",
      "timerDuration": "20s",
      "buttonCommands": [
          {
              ...
          },
          ...
      ],
      "timerCommands": [
        {
            ...
        },
        ...
      ]
  }
}

{
  "notification.remove": {
      "id": "sound"
  }
}
