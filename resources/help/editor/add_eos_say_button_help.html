Add Say Instruction

This button adds a say instruction to the currently opened eos script in the EOS Editor.


The following parameters can be changed for choice instructions:

  • label (json attr: label): This is the text shown in the speech bubble. Can be html and can contain evaluator expressions: <eval>//code here</eval>.
  • align (json attr: align): The alignment of the text bubble.
  • mode (json attr: mode): The play mode of the text.
    • autoplay: The next instruction will be triggered after duration time has passed or if the player clicks on the screen (if allow skip is enabled).
    • instant: The next instruction will be executed immediately ignoring the wait time.
    • pause: Text will be shown until the user clicks it away.
    • custom: allow skip and duration will be ignored.
  • allow skip (json attr: allowSkip): This flag enables the text to be skipped by clicking the screen.
  • duration (json attr: allowSkip): The duration for which the text bubble is shown. If left empty or set to -1 the duration will be determined automatically based on the content. Needs to follow the format <weeks>w<days>d<hours>h<minutes>m<seconds>s<milis>ms or be a plain number denoting miliseconds. For example: 2m30s or 100.


Dialogue UI

A full node in serialised JSON form:

{
    "say": {
        "label": "myText",
        "align": "center",
        "mode": "autoplay",
        "allowSkip": true,
        "duration": "20s"
    }
}
