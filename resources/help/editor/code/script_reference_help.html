Script Reference

Below you can find a full script interface reference for the custom JavaScript objects registered by the JOIP-Engine. For general JavaScript reference please see:

  • Qt-Script Reference: https://doc.qt.io/qt-5/ecmascript.html
  • JavaScript Help: https://www.w3schools.com/js/


Javascript Objects are registered in the Player by the currently loaded Layout. The name of the registered objects is defined in the layout file. In the reference below, the default names of the registered objects are listed.


PlayerBackground (default name: background)


This object is automatically registered by the JOIP-Engine on loading a JOI-Project but can also be placed in the Layout manually. The default object is allways registered under the name background and spans the entire background. It only has a few callable functions:

  • setBackgroundColor ( string | int[3 | 4] ):
    As the name implies, this will set the background color, which is overlayed over the background texture. The given argument can be an html-style color string ("#argb") or an array of 3 or 4 numbers ranging from 0 - 255 representing red, green, blue and alpha in this order.
    Example Usage:
    background.setBackgroundColor("#88ff0000"); /* will set background to a 50% transparent red */
    background.setBackgroundColor("red"); /* will set background to a 0% transparent red */
    background.setBackgroundColor([255, 0, 0, 128]); /* will set background to a 50% transparent red */

  • setBackgroundTexture ( string | Resource | null ):
    As the name implies, this will set the background texture. The texture is repeated as often as neccessary to fill the entire object. The argument can either be the name of a Resource or a resource object obtained from the current Scene object. In any case, it has to be of the Image type. A null argument will reset the background to the style-defined application background.
    Example Usage:
    background.setBackgroundTexture("tiledBg");
    background.setBackgroundTexture(null);
    background.setBackgroundTexture(bgObject); /* note: in this example bgObject is a var */


PlayerIcons (default name: icon)


This object is registered by the JOIP-Engine if it is present in the current Layout. It is a container for a list of icons that can be shown or hidden at will.

  • show ( string | Resource ):
    This function will add a new icon to the list, where the resource argument will be the shown icon. The Resource has to be of the Image or Movie type and can either be the name of a Resource or a resource object obtained from the current Scene object.
    Example Usage:
    icon.show("myIcon");
    icon.show(myIconVar); /* note: in this example myIconVar is a var */

  • hide ( [ string ] ):
    This function will remove the icon with the name given in the input. An empty input, no input or the input string: "~all" will remove all icons.
    Example Usage:
    icon.hide("myIcon"); // hides myIcon
    icon.hide(); // hides all icons
    icon.hide(""); // hides all icons
    icon.hide("~all"); // hides all icons


PlayerMediaPlayer (default name: mediaPlayer)


This object is registered by the JOIP-Engine if it is present in the current Layout. This is the main way to display or play media Resources of types Image, Movie or Sound.

  • show ( string | Resource ):
    This function shows an Image type Resource. Calling this will stop any currently running playback. The Resource can be the name of a Resource or a Resource object obtained from the current Scene object.
    Example Usage:
    mediaPlayer.show("myIcon");
    mediaPlayer.show(myMediaVar); /* note: in this example myMediaVar is a var */

  • play ( [ string | Resource ] ):
    This function can be called with or without arguments. Calling it with arguments plays a Movie or Sound type Resource. When doing this, it will stop any currently running playback and start replay of the new media. The Resource can be the name of a Resource or a Resource object obtained from the current Scene object. When calling this without any argument, it will continue playing the last paused or stopped sound and video.
    Example Usage:
    mediaPlayer.play("myVideoOrSound"); // plays myVideoOrSound
    mediaPlayer.play(myVideoOrSoundVar); /* note: in this example myVideoOrSoundVar is a var */
    mediaPlayer.play(); // continues playing the last played media
  • playVideo ( ):
    This function does the same as if you call play without arguments, but will allways continue playing the last paused or stopped video instead of all media.
    Example Usage:
    mediaPlayer.playVideo(); // continues playing the last played video

  • pauseVideo ( ):
    This function will pause the currently playing video Resource. A paused video can never stop the wait in waitForVideo or waitForPlayback since the video is not considered finished yet.
    Example Usage:
    mediaPlayer.pauseVideo(); // pauses playing the last played video

  • stopVideo ( ):
    This function will stop the currently playing video Resource.
    Example Usage:
    mediaPlayer.stopVideo(); // stops playing the last played video

  • playSound ( ):
    This function does the same as if you call play without arguments, but will allways continue playing the last paused or stopped sound instead of all media.
    Example Usage:
    mediaPlayer.playSound(); // continues playing the last played sound

  • pauseSound ( ):
    This function will pause the currently playing sound Resource. A paused sound can never stop the wait in waitForSound or waitForPlayback since the sound is not considered finished yet.
    Example Usage:
    mediaPlayer.pauseSound(); // pauses playing the last played sound

  • stopSound ( ):
    This function will stop the currently playing sound Resource.
    Example Usage:
    mediaPlayer.stopVideo(); // stops playing the last played video

  • waitForPlayback ( ):
    Blocks and waits until either the currently running video or sound Resource stopps playing. If no sound or video is playing this function will return immediately. If both sound and video are paused this function will never return.
    Example Usage:
    mediaPlayer.waitForPlayback();

  • waitForVideo ( ):
    Blocks and waits until the currently running video Resource stopps playing. If no video is playing this function will return immediately. If the video Resouce is paused this function will never return.
    Example Usage:
    mediaPlayer.waitForVideo();

  • waitForSound ( ):
    Blocks and waits until the currently running sound Resource stopps playing. If no sound is playing this function will return immediately. If the sound Resouce is paused this function will never return.
    Example Usage:
    mediaPlayer.waitForVideo();


PlayerStorage (name: localStorage)


This object is automatically registered by the JOIP-Engine on loading a JOI-Project and should not be placed in the Layout manually. Similarly to the local storage of a browser, this object stores values that can be retrieved at any time. The storage will remain until the currently running JOI-Project finishes playing.

  • store ( string id, var value ):
    Stores a value with the given id in the storage. Any value can be stored and loaded, even complex objects.
    Example Usage:
    localStorage.store("value", 5);

  • load ( string id ): value:
    Loads a value with the given id from the storage. Any value can be stored and loaded, even complex objects.
    Example Usage:
    var value = localStorage.load("value");


PlayerTextBox (default name: textBox)


This object is registered by the JOIP-Engine if it is present in the current Layout. This is the main way to display text or recieve player input.

  • setBackgroundColors ( string[] | int[][3 | 4] ):
    As the name implies, this will set the background color of elements in the text box. The given argument can be an array of html-style color strings ("#argb") or an array of arrays of 3 or 4 numbers ranging from 0 - 255 representing red, green, blue and alpha in this order. The input takes an array of colors, since multiple buttons can be shown simultaniously and those need to be able to be colored individually.
    Example Usage:
    textBox.setBackgroundColors(["#88ff0000", "red"]); /* will set background of text to a 50% transparent red. In case of two buttons the background of the first one will be 50% transparent red and the second will be 0% transparent red */
    textBox.setBackgroundColors([[255, 0, 0, 128], [255, 0, 0]]); /* same as above example */
    textBox.setBackgroundColors([[255, 0, 0, 128], "red"]); /* same as above example */

  • setTextAlignment ( enum:TextAlignment ):
    This will set the text alignment of texts resulting from showText calls in the text box. Once set, all future showText calls will display text with the set alignment. The given argument needs to be one of the enum values: TextAlignment.AlignLeft, TextAlignment.AlignRight, TextAlignment.AlignCenter or their coresponding nummerical values (1, 2, 4 respectively).
    Changing the alignment will change the position of portraits set by setTextPortrait.
    This will not influence calls to showButtonPrompts or showInput.
    Example Usage:
    textBox.setTextAlignment(TextAlignment.AlignLeft); /* text will be left aligned with portraits to the right */
    textBox.setTextAlignment(1); /* same as above */
    textBox.setTextAlignment(TextAlignment.AlignRight); /* text will be right aligned with portraits to the left */
    textBox.setTextAlignment(2); /* same as above */
    textBox.setTextAlignment(TextAlignment.AlignCenter); /* text will be left aligned with portraits centered above the text */
    textBox.setTextAlignment(4); /* same as above */

  • setTextColors ( string[] | int[][3 | 4] ):
    As the name implies, this will set the text color of elements in the text box. The given argument can be an array of html-style color strings ("#argb") or an array of arrays of 3 or 4 numbers ranging from 0 - 255 representing red, green, blue and alpha in this order. The input takes an array of colors, since multiple buttons can be shown simultaniously and those need to be able to be colored individually.
    Example Usage:
    textBox.setTextColors(["#ffff0000", "green"]); /* will set text color to a 0% transparent red. In case of two buttons the text color of the first one will be 0% transparent red and the second will be 0% transparent green */
    textBox.setTextColors([[255, 0, 0, 255], [0, 255, 0]]); /* same as above example */
    textBox.setTextColors([[255, 0, 0, 255], "green"]); /* same as above example */

  • setTextPortrait ( string | Resource | null ):
    This will set an Image or Movie type Resource to be shown besides text resulting from showText calls. The Resource can be the name of a Resource, a Resource object obtained from the current Scene object or null. Calling the function with null will remove the portraits from future texts resulting from showText calls.
    Example Usage:
    textBox.setTextPortrait("myIcon");
    textBox.setTextPortrait(myIconVar); /* note: in this example myIconVar is a var */

  • showButtonPrompts ( string[] ):
    Shows a set of buttons. The array of input strings will be used for the button texts and the colors set by setBackgroundColors and setTextColors will be used for text and background coloring. This function blocks until the player clicks a button, upon which the index of the pressed button will be returned (the returned values are 0 indexed). It is possible to use a subset of html tags in the buton texts (see https://doc.qt.io/qt-5/richtext-html-subset.html for the supported subset).
    Example Usage:
    var choice = textBox.showButtonPrompts(["Yes","No"]); /* Will return 0 if the player presses "Yes" and 1 if the player presses "No". */

  • showInput ( ): string:
    Shows a text input editor. The colors set by setBackgroundColors and setTextColors will be used for text and background coloring. This function blocks until the player enters a text, upon which the text will be returned.
    Example Usage:
    var input = textBox.showInput();

  • showText ( string ):
    Shows text. The input string will be used for the text content and the first colors set by setBackgroundColors and setTextColors will be used for text and background coloring. It is possible to use a subset of html tags in the text (see https://doc.qt.io/qt-5/richtext-html-subset.html for the supported subset).
    Example Usage:
    textBox.showText("Hello world!");
    textBox.showText("Hello<br>world!"); /* Will insert a newline between Hello and world! */

  • clear ( ):
    Clears the log of the textbox. This will not clear any previously set colors.
    Example Usage:
    textBox.clear();


PlayerTimer (default name: timer)


This object is registered by the JOIP-Engine if it is present in the current Layout. This object is for displaying a timer that can be shown or hidden on demand.

  • hide ( ):
    Hides the timer completely, but does not stop any running timer.
    Example Usage:
    timer.hide();

  • setTime ( int ):
    Sets the counter of the timer to the input. The input is given in seconds. Setting the time during a running timer will reset the timer.
    Example Usage:
    timer.setTime(120); /* Sets the timer to 2 min */

  • setTimeVisible ( bool ):
    Sets the counter of the timer visible or invisible. A timer with an invisible counter will display a question mark instead of the remaining time. This setting will persist, even if the timer ends.
    Example Usage:
    timer.setTimeVisible(true);

  • show ( ):
    Shows the timer, but does not stop any running timer.
    Example Usage:
    timer.show();

  • start ( ):
    Starts the timer.
    Example Usage:
    timer.start();

  • stop ( ):
    Stops the timer.
    Example Usage:
    timer.stop();

  • waitForTimer ( ):
    Blocks and waits until the timer finishes. If no timer is running, this function will return immediately.
    Example Usage:
    timer.waitForTimer();


PlayerThread (name: thread)


This object is automatically registered by the JOIP-Engine on loading a JOI-Project and should not be placed in the Layout manually. This object is only used to idle the script for a certain time.

  • sleep ( int timeout [, bool skippable ] ):
    Blocks and waits for timeout seconds. Skippable is optional and if it is set to true, the wait can by skipped by the player.
    Example Usage:
    thread.sleep(10,true); /* Skippable 10s wait */
    thread.sleep(10,false); /* Unskippable 10s wait */
    thread.sleep(10); /* Unskippable 10s wait */


Project


A reference to the current Project object can be retrieved from the currently playing Scene object. This object can be used to retrieve resources and data about the current JOI-Project. This object should not be placed in the Layout manually.

  • id: int:
    This property lets you read the internal Project id.
  • version: int:
    This reads the project version. Useful, if you need to have your scripts be compatible with different JOIP-Engine versions. The version is a number with bits 16-23 being major, bits 8-15 being minor and the last eight being the patch version (0x00MMmmPP).
    Example Usage:
    var major = (project.version & 0x00ff0000) >> 16; /* read major version */
    var minor = (project.version & 0x0000ff00) >> 8; /* read minor version */
    var path = project.version & 0x000000ff; /* read path version */
  • versionText: string:
    This property reads the project version as a formated user-readable string.
  • targetVersion: int:
    This reads the target engine version. Useful, if you need to have your scripts be compatible with different JOIP-Engine versions. The version is a number with bits 16-23 being major, bits 8-15 being minor and the last eight being the patch version (0x00MMmmPP).
    Example Usage:
    var major = (project.version & 0x00ff0000) >> 16; /* read major version */
    var minor = (project.version & 0x0000ff00) >> 8; /* read minor version */
    var path = project.version & 0x000000ff; /* read path version */
  • targetVersionText: string:
    This property reads the target engine version as a formated user-readable string.
  • name: string:
    This property lets you read the Project name.
  • folderName: string:
    This property lets you read the Project's folder name.
  • describtion: string:
    This property lets you read the Project's describtion.
  • titleCard: string:
    This property lets you read the Resources name representing the title card of the JOI-Project.
    Example Usage:
    mediaPlayer.show(project.titleCard);
  • map: string:
    This property lets you read the Resources name representing the map of the JOI-Project. Because this is a currently unused and not fully implemented functionality, this property shouldn't be accessed.
  • sceneModel: string:
    This property lets you read the Resources name representing the scene model of the JOI-Project. This property is only useful for manually parsing the file.
  • playerLayout: string:
    This property lets you read the Resources name representing the Layout file of the JOI-Project. This property is only useful for manually parsing the file.
  • kinks: string[]:
    This property lets you read the kinks, that the application has been tagged with.
  • isUsingWeb: bool:
    This property lets you read the using-web-flag that is set, if the project contains any remote Resources.
  • isUsingCodecs: bool:
    This property lets you read the using-codecs-flag that is set, if the project contains any video or sound Resources.
  • isBundled: bool:
    This property lets you read the bundled-flag that is set, if the project was exported and is thus read-only.
  • numScenes ( ): int:
    Returns the number of scenes in the project. Usefull for iteration.
  • scenes ( ): string[]:
    Returns all scene names. Useful for iteration.
  • scene ( int index | string name ): Scene:
    Returns a Scene object at index index or with the name name. Useful if you need to read scene specific properties.
    Example Usage:
    var myScene = project.scene("myScene");
    var myScene = project.scene(0);
  • numResources ( ): int:
    Returns the number of Resources in the project. Usefull for iteration.
  • resources ( ): string[]:
    Returns all Resource names. Useful for iteration.
  • resource ( int index | string name ): Resource:
    Returns a Resource object at index index or with the name name. Useful if you need to read Resource specific properties.
    Example Usage:
    var myResource = project.resource("myScene");
    var myResource = project.resource(0);


Scene (name: scene)


This object is automatically registered by the JOIP-Engine on loading a JOI-Project and should not be placed in the Layout manually. This object should not be placed in the Layout manually.

  • id: int
    This property lets you read the internal Scene id.
  • name: string:
    This property lets you read the Scene name.
  • script: string
    Gets the script file name that is associated with the scene.
  • numResources ( ): int:
    Returns the number of Resources in the project. Currently unused and will allways return 0.
  • resources ( ): string[]:
    Returns all Resource names. Currently unused and will allways return an empty list.
  • resource ( string name ): Resource:
    Returns a Resource object with the name name. Currently unused and will allways return undefined.
  • project ( ): Project:
    Returns the Project object that contains this Scene. Useful for getting information about the Project itself, or about Resources.
    Example Usage:
    var project = scene.project();


Resource


This type of object can be returned by a Project object and contains information about the Resource. It can also be used as argument in the PlayerBackground, PlayerIcons, PlayerMediaPlayer etc. This object should not be placed in the Layout manually.

  • isAnimated: bool
    This property lets you read if the resource is animated (gif image or video Resource). An animated Resource can be played or paused and video Resources can be waited on with waitForVideo or waitForPlayback.
  • isLocal: bool
    This property lets you read if the Resource is local or online. For online resources, the path points to the image or video url directly and for local resources it contains the local file relave to the JOI-Project root folder.
  • name: string
    This property lets you read the Resource name. This also serves as the Resource identifier.
  • path: url
    This property lets you read the path to the Resource.For online resources, the path points to the image or video url directly and for local resources it contains the local file relave to the JOI-Project root folder.
  • type: ResourceType
    This property lets you read the resource type. The values are:
    ResourceType.Image (0): Image Resource
    ResourceType.Movie (1): Movie Resource
    ResourceType.Sound (2): Sound Resource
    ResourceType.Other (3): Flow file, Project file, Scene Script files and Layout file
  • project ( ): Project:
    Returns the Project object that contains this Resource. Useful for getting information about the Project itself, or about other Resources or Scenes.
    Example Usage:
    var project = resource.project();


return


While not an actual object, this JavaScript language feature is used to "skip" to a Scene directly without a normal Scene Transition. The value to be returned is the Scene name as a string.
Example Usage:
return "SceneB"; // skipps to "SceneB" and terminates current script
