Export Project

Saves the current JOI-Project in a bundled format, which creates a .proj file in the project directory. This file is named after the project and contains all project resources in a single bundled binary file. This file can then be distributed instead of the loose files in the project folder. Performing this action will not delete the opened project nor change any of it's properties, as the exported file is only a copy of the project.


Exporting a project has the important advantage of making the project read only (when loaded via the .proj file instead of the loose files), preventing other users from easily copying files from the project or making modifications to it.



Due to an exported project being in a binary format, they currently need to be loaded into memory all at once.
This means exported projects should be kept to a reasonable size, as too big projects will not be loadable by every pc.
