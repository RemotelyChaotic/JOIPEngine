Node

A JOI-Project consists of nodes of different types, that dictate the flow of a project in the player.
Nodes have their own settings and a set of connectors on the sides. There's five different types of nodes:

  • Entry Point: The start of a JOI-Project in the player is always an "Entry Point" node. There must always be exactly one "Entry Point" node in a Project, otherwise the behavior is undefined and the Project can start at any "Entry Point" node.
    This is not a scene, and no script is associated with an "Entry Point" node.
  • Exit Point: The end of a JOI-Project in the player is always an "Exit Point" node. There's no limit on the number of "Exit Point" nodes per Project. One thing to consider is: every path in the project should always eventually lead to an "Exit Point".
    This is not a scene, and no script is associated with an "Exit Point" node.
  • Scene: Scene nodes represent logical units, that contain a set of instructions that the player will execute. When you create a "Scene" node, you can assign the associated script Resource file (it helps to have the same name as the "Scene" node, but it's not neccessary; just make sure it's in the project folder.)
    It is also possible to definte a title resource and set if it is possible to start the tease at thise scene. If enabled, the scene selection will list this scene with the selected reesource as preview.
    There's no limit on the number of "Scene" nodes in a project.
    The line edit shows the internal id of a scene. This is not user visible, must be unique, and can be used in Scripts to jump to the scene ignoring the flow.
  • Path Splitters: These nodes split the flow of a project into up to four possible paths. Depending on the type of the "Path Splitter" node the flow in the player will either continue with a random node, or prompt the user to select the path.
    By selecting &qout;Button Labels&qout; and putting the splitter into select mode, a custom transition can be used if needed. This transition uses a Layout Resource to display the selection UI. To create a new transition from a template, click the button next to the Resource selector.
    "Path Splitter" and "Path Merger" nodes can be combined and chained freely to create more path alternatives and a mix between random and selectable flow. Any empty connectors will be ignored during playing and you do not need to worry about them.
    The four line edits are for the user visible path names if the node is of type selectable. They corespond to the user visible button labels in the editor and the labels available to custom transitions (variable: vsInput).
  • Path Merger: These nodes merge the flow of a project from up to four possible paths.
    "Path Splitter" and "Path Merger" nodes can be combined and chained freely to create more path alternatives and a mix between random and selectable flow. Any empty connectors will be ignored during playing and you do not need to worry about them.
