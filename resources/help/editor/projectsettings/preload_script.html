Preload Script

Lets you set a Script that is executed before loading the JOI-Project by the Player. The script can be a Script Resource of type js or lua. This Script allows you to modify the Project before loading it.


Clicking the button will open an interface to add a new Script file.


This Preload Scipt is useful for example if properties like the Project Layout need to be Engine Version dependant because of underlying QML API incompatibilities.


The following objects and properties can be accessed in this script:


Project (name: project)


A reference to the current Project to be loaded. This object is automatically registered by the JOIP-Engine on loading the Script. This object can be used to retrieve and modify resources and data of the current JOI-Project.

  • id: int: (r)
    This property lets you read the internal Project id.

  • version: int: (r/w)
    This reads the project version. Useful, if you need to have your scripts be compatible with different JOIP-Engine versions. The version is a number with bits 16-23 being major, bits 8-15 being minor and the last eight being the patch version (0x00MMmmPP).
    Example Usage (JS):
    var major = (project.version & 0x00ff0000) >> 16; /* read major version */
    var minor = (project.version & 0x0000ff00) >> 8; /* read minor version */
    var path = project.version & 0x000000ff; /* read path version */
    Example Usage (Lua):
    local major = (project.version & 0x00ff0000) >> 16; -- read major version
    local minor = (project.version & 0x0000ff00) >> 8; -- read minor version
    local path = project.version & 0x000000ff; -- read path version

  • versionText: string: (r/w)
    This property reads the project version as a formated user-readable string.
  • targetVersion: int: (r/w)
    This reads the target engine version. Useful, if you need to have your scripts be compatible with different JOIP-Engine versions. The version is a number with bits 16-23 being major, bits 8-15 being minor and the last eight being the patch version (0x00MMmmPP).
    Example Usage:
    var major = (project.version & 0x00ff0000) >> 16; /* read major version */
    var minor = (project.version & 0x0000ff00) >> 8; /* read minor version */
    var path = project.version & 0x000000ff; /* read path version */
  • targetVersionText: string: (r/w)
    This property reads the target engine version as a formated user-readable string.

  • name: string: (r)
    This property lets you read the Project name.

  • folderName: string: (r)
    This property lets you read the Project's folder name.

  • description: string: (r/w)
    This property lets you read the Project's description.

  • titleCard: string: (r/w)
    This property lets you read the Resources name representing the title card of the JOI-Project.
    Example Usage (JS):
    mediaPlayer.show(project.titleCard);
    Example Usage (Lua):
    mediaPlayer:show(project.titleCard);

  • map: string: (r/w)
    This property lets you read the Resources name representing the map of the JOI-Project. Because this is a currently unused and not fully implemented functionality, this property shouldn't be accessed.

  • sceneModel: string: (r/w)
    This property lets you read the Resources name representing the scene model of the JOI-Project. This property is only useful for manually parsing the file.

  • playerLayout: string: (r/w)
    This property lets you read the Resources name representing the Layout file of the JOI-Project.

  • preLoadScript: string (r):
    This property lets you read the Resource name representing this Preload Script file of the JOI-Project.

  • numberOfSoundEmitters: int (r/w):
    This property lets you read the Number of Sound Emitters instaced by the Engine.
  • metCmdMode: int (r/w)
    The Metronome Commands sent to the DeviceController by the PlayerMetronome. Possible flag values are the following (flag values may be OR-ed together):
    • 0: No commands are sent
    • 1: Send the vibrate command
    • 2: Send the linear command
    • 4: Send the rotate command
    • 7: Send all commands


  • canStartAtAnyScene: bool (r)
    This property lets you read the canStartAtAnyScene-flag that can be set to indicate, if the project can be started at any Scene.

  • isUsingWeb: bool: (r)
    This property lets you read the using-web-flag that is set, if the project contains any remote Resources.

  • isUsingCodecs: bool: (r)
    This property lets you read the using-codecs-flag that is set, if the project contains any video or sound Resources.

  • isBundled: bool: (r)
    This property lets you read the bundled-flag that is set, if the project was exported and is thus read-only.

  • isReadOnly: bool: (r)
    This property lets you read the read-only-flag that is set if the project was loaded as read-only due to being an archive.

  • dlState: DownloadState: (r)
    This property lets you read the download-state of the project. The download states are Unstarted, DownloadRunning, Finished. A playable project will always have this set to Finished and you will probably never need to check this property.

  • numKinks ( ): int: (r)
    This property lets you read the number of kinks, that the application has been tagged with.

  • kinks: string[]: (r)
    This property lets you read the kinks, that the application has been tagged with.

  • kink ( string name ): Kink: (r)
    Returns a Kink object with the name name. Useful if you need to read Kink specific properties.
    Example Usage (JS):
    var myScene = project.kink("Vaginal Sex (Giving)");
    Example Usage (Lua):
    local myScene = project:kink('Vaginal Sex (Giving)');

  • numScenes ( ): int: (r)
    Returns the number of scenes in the project. Usefull for iteration.

  • scenes ( ): string[]: (r)
    Returns all scene names. Useful for iteration.

  • scene ( int index | string name ): Scene: (r)
    Returns a Scene object at index index or with the name name. Useful if you need to read scene specific properties.
    Example Usage (JS):
    var myScene = project.scene("myScene");
    var myScene = project.scene(0);
    Example Usage (Lua):
    local myScene = project:scene('myScene');
    local myScene = project:scene(0);

  • numResources ( ): int: (r)
    Returns the number of Resources in the project. Usefull for iteration.

  • resources ( ): string[]: (r)
    Returns all Resource names. Useful for iteration.

  • resource ( int index | string name ): Resource: (r)
    Returns a Resource object at index index or with the name name. Useful if you need to read Resource specific properties.
    Example Usage (JS):
    var myResource = project.resource("myScene");
    var myResource = project.resource(0);
    Example Usage (Lua):
    local myResource = project:resource('myScene');
    local myResource = project:resource(0);

  • numTags ( ): int: (r)
    Returns the number of Tags in the project. Usefull for iteration.

  • tags ( ): string[]: (r)
    Returns all Tag names. Useful for iteration.

  • tag ( string name | int index ): Tag: (r)
    Returns a Tag object at index index or with the name name. Useful if you need to use the Tag for custom logic.

  • numAchievements ( ): int: (r)
    Returns the number of Achievements in the project. Usefull for iteration.

  • achievements ( ): string[]: (r)
    Returns all Achievements names. Useful for iteration.

  • achievement ( string name | int index ): SaveData: (r)
    Returns a SaveData object representing an achievement at index index or with the name name. Useful if you need to use the Achievement for custom logic.

  • font: string: (r/w)
    Returns the font used in the player. Mainly in the PlayerTextBox.


Settings (name: Settings)


A global property of the script set by the Engine that allows access to the Engine settings. All of its properties are read-only.

  • autoUpdate: bool
    Checks if autoupdates are enabled.

  • connectToHWOnStartup: bool
    Flag that tells the engine to connect to hardware interfaces on startup.

  • contentFolder: string
    The Data Folder of the engine. Basically unusable since System paths can not be accessed directly.

  • debugOverlayEnabled: bool
    checks if the Debug Overlay is enabled.

  • dominantHand: int
    the Dominant Hand setting. Possible values are:
    • 0: No dominant hand set
    • 1: Left handed use
    • 2: Right handed use


  • editorCaseInsensitiveSearch: bool
    Denotes if editor search in case in-sensitive or case-sensitive (caution: the property is true if the case is in-sensitive. See: Editor Settings.

  • editorFont: string
    The Font of the editor. See: Editor Settings.

  • editorShowWhitespace: bool
    Flag if whitespace is visualised in the editor. See: Editor Settings.

  • editorTheme: string
    Theme of the editor. See: Editor Settings.

  • font: string
    The used Engine Font.

  • keyBindings: Array [ string ]
    The list of keybinding roles of the editor.

  • keyBinding( string ): string
    The keybinding of a given keybinding role.

  • hideSettingsTimeout: int
    The Timeout in ms for automatically hiding the textBox in the Player.

  • metronomeDefaultCommands: int
    The Default Metronome Commands sent to the DeviceController by the PlayerMetronome. Possible flag values are the following (flag values may be OR-ed together):
    • 0: No commands are sent
    • 1: Send the vibrate command
    • 2: Send the linear command
    • 4: Send the rotate command
    • 7: Send all commands


  • metronomeSfx: string
    The default audio file id for the metronome. See Metronome Settings

  • metronomeSizeRel: real
    The relative size of the metronome to the Player frame. See Metronome Size Settings

  • metronomeSizeMin: int
    the absolute minimal height of the metronome in pixels. See Metronome Size Settings

  • metronomeVolume: real
    the metronome volume from 0 to 1. See Metronome Settings

  • muted: bool
    The Muted Engine setting flag.

  • offline: bool
    The Offline Mode Engine setting flag. In offline mode, remote Resources will not be loaded.

  • pauseWhenInactive: bool
    The Pause Player when app is not active Engine setting flag.

  • platform: string
    The platform string the Engine is running on crrently. Is one of:
    • Android: Running on Android devices
    • Windows: Running on Windows OS
    • Linux: Running on any Linux OS except Android


  • playerAntialiasing: bool
    Flag if anti-aliasing in the player is active. See Player Settings

  • playerDropShadow: bool
    Flag if drop-shadows in the player are active. See Player Settings

  • playerImageMipMap: bool
    Flag if mip-mapping for Images in the player is active. See Player Settings

  • playerImageSmooth: bool
    Flag if smooth scaling for Images in the player is active. See Player Settings

  • preferedEditorLayout: int
    The currently set Editor Layout. Is one of:
    • eNone (0): Not set yet. The default (eModern) is used.
    • eClassic (1): The old Layout split into two sides.
    • eModern (2): The new Layout split into two sides with the Resource Manager always at the bottom.
    • eCompact (3): the compact mobile layout with only one open view.


  • pushNotifications: bool
    Checks if Push Notifications are active.

  • resolution: size
    The Window Resolution of the Engine.

  • style: string
    The Style of the Engine.

  • styleHotLoad: bool
    The Style Hotload flag.

  • version: int
    The Engine Version.

  • volume: real
    The global Volume of the Engine.

  • windowMode: int
    the Engine window's Window Mode.
    Is one of:
    • eFullscreen (0): Application fills a screen.
    • eBorderless (1): Application respects the selected Resolution but is displayed without a header.
    • eWindowed (2): Application is shown as moveable, resizable window and respects the selected Resolution.



Scene


This type of object can be returned by a Project object and contains information about the Scene.

  • id: int (r)
    This property lets you read the internal Scene id.

  • name: string (r):
    This property lets you read the Scene name.

  • script: string (r/w)
    Gets the script file name that is associated with the scene.

  • sceneLayout: string (r/w)
    Gets the Layout resource file that should be set when the scene is loaded.
    This is not yet implemented, but planned for future versions.
  • numResources ( ): int:
    Returns the number of Resources in the project. Currently unused and will always return 0 unless scenes have been added manually.

  • resources ( ): string[]:
    Returns all Resource names. Currently unused and will always return an empty list unless scenes have been added manually.

  • resource ( string name ): Resource:
    Returns a Resource object with the name name. Currently unused and will always return undefined unless scenes have been added manually.

  • addResource ( string name ):
    Adds a Resource with the name name to the scene to be retrieved later.

  • removeResource ( string name ):
    Removes a resource with the name name from the scene. Currently unused and will not work unless scenes have been added manually.

  • project ( ): Project:
    Returns the Project object that contains this Scene. Useful for getting information about the Project itself, or about Resources.
    Example Usage (JS):
    var project = scene.project();
    Example Usage (Lua):
    local project = scene:project();


Resource


This type of object can be returned by a Project object and contains information about the Resource.

  • isAnimated: bool (r):
    This property lets you read if the resource is animated (gif image or video Resource). An animated Resource can be played or paused and video Resources can be waited on with waitForVideo or waitForPlayback.
  • isLocal: bool (r):
    This property lets you read if the Resource is local or online. For online resources, the path points to the image or video url directly and for local resources it contains the local file relave to the JOI-Project root folder.
  • name: string (r):
    This property lets you read the Resource name. This also serves as the Resource identifier.
  • path: url (r):
    This property lets you read the path to the Resource.For online resources, the path points to the image or video url directly and for local resources it contains the local file relave to the JOI-Project root folder.
  • source: url (r):
    This property lets you read the source of the Resource.
    This is not the file path, but the origin where the resource was found (for credit purposes).
  • type: ResourceType (r):
    This property lets you read the resource type. The values are:
    ResourceType.Image (0): Image Resource
    ResourceType.Movie (1): Movie Resource
    ResourceType.Sound (2): Sound Resource
    ResourceType.Other (3): Flow file, Project file, Scene Script files and Layout file
  • numTags ( ): int:
    Returns the number of Tags of the Resource. Usefull for iteration.

  • tags ( ): string[]:
    Returns all Tag names of the Resource. Useful for iteration.

  • tag ( string name | int index ): Tag:
    Returns a Tag object at index index or with the name name. Useful if you need to use the Tag for custom logic.

  • addTag ( string name, string category, string description ):
    Adds a new Tag object at with the name, category and description given. Useful if you need to use the Tag for custom logic.

  • removeTag ( string name | int index ):
    Removes a Tag object at index index or with the name name. Useful if you need to use the Tag for custom logic.

  • project ( ): Project:
    Returns the Project object that contains this Resource. Useful for getting information about the Project itself, or about other Resources or Scenes.
    Example Usage (JS):
    var project = resource.project();
    Example Usage (Lua):
    local project = resource:project();


Tag


This type of object can be returned by a Project object and contains information about the specific Tag registered in the database.

  • type: string (r):
    This property lets you read the Tag category.
  • name: string (r):
    This property lets you read the Tag name. This also serves as the Tag identifier.
  • description: string (r):
    This property lets you read the description of the Tag.
  • color ( ): color (r):
    This property lets you read the color of the Tag that is used in the UI. The red green and blue values are generated from an MD4 Hash of the category where red is the first byte green the second and blue the third.
  • numResources ( ): int
    The number of resources that are assigned with this tag.
  • resources ( ): string[]
    The resource names that are assigned with this tag.


SaveData


This type of object can be returned by a Project object and contains information about an Achievement. The type name is SaveData, because this is a generic persistent Data wrapper.

  • name: string (r):
    This property lets you read the name / ID of the element. This serves as the identifier in the project and in any persistent file.
  • description: string (r/w):
    This property lets you read the description of the SaveData if there is any.
  • type: string (r):
    This property lets you read the SaveData type. Only Bool and Int are supported by Achievements. The possible types are:
    • SaveDataType.Bool: Contains a boolean value.
    • SaveDataType.Int: Contains an integer 32bit value.
    • SaveDataType.Double: Contains a double (real) 64bit value.
    • SaveDataType.String: Contains a string object.
    • SaveDataType.Regexp: Contains a regular expression.
    • SaveDataType.Date: Contains a Date object.
    • SaveDataType.Url: Contains an Url object.
    • SaveDataType.Array: Contains an array of elements.
    • SaveDataType.Object: Contains a complex JSON object.
    • SaveDataType.Null: Contains a null object.
  • resource: string (r/w):
    The number of resource that is assigned to this SaveData element. In case of an Achievement this is the icon.
  • data: var (r/w):
    The resource data. Type depends on the type of the SaveData.


Kink


This type of object can be returned by a Project object and contains information about the specific Kink registered in the database. It is used internally to visualize the kinks associated with a Project.

  • idForOrdering: int (r):
    This property lets you read the internal index of the Kink, which is used for sorting.
  • type: string (r):
    This property lets you read the kink category.
  • name: string (r):
    This property lets you read the Kink name. This also serves as the Kink identifier.
  • description: string (r):
    This property lets you read the description of the Kink.
  • color ( ): color
    This property lets you read the color of the Kink that is used in the UI. The red green and blue values are generated from an MD4 Hash of the category where red is the first byte green the second and blue the third.
