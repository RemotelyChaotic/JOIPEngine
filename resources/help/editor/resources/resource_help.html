Resource

Resources are the content files of a JOI-Project and must reside in the folder directory. Files outside the project directory cannot be used. Resources can represent files of different formats and types and are referenced in the application by their internal name / identifier, which does not need to equal the file name.


Resources can be renamed by double-clicking the name field in the Resource Tree. Renaming a resource changes it's internal identifier, but not the file name.


There's currently six differnt Resource types:

  • Image: Images can be animated or static and are used for the Title Card and to display in the player via Media Player or as an Icon. Images can be online or offline. For online Resources, the link must point to the image or video url directly. Links gated behind logins will not work. These Resources will not be saved on your disk and instead (down-)loaded into memory and displayed when needed.
    Supported image formats are: bmp, cur, gif, icns, ico, jpeg, jpg, pbm, pcx, pgm, png, ppm, svg, svgz, tga, tif, tiff, wbmp, webp, xbm, xpm
  • Video: Videos can be displayed in the player via Media Player. Videos can be online or offline. Online videos only support a subset of the formats. For online Resources, the link must point to the image or video url directly. Links gated behind logins will not work. These Resources will not be saved on your disk and instead streamed into memory and displayed when needed.
    Supported video formats are: 3gp, asf, wmv, mpg, ts, au, avi, flv, mov, mp4 (online available), ogm, ogg, mkv, mka, nsc, nsv, nut, a52, dv, vid, ty, webm (online available)
  • Audio: Audio files can be played in the player via Media Player.
    Supported audio formats are: wav, dts, mp3, mp2, tta, tac, flac, dts, aac, xa
  • Scripts: This Resource type is used for the Scene Scripts. They can be opened in any external text editor to edit the contents.
    Supported script formats are: js
  • Fonts: This Resource type is used in the Scene Scripts to set the font-family of a block of text to a custom font. The name of the resource is not used anywhere and is only for flavour and consistency. To use the custom fonts you need to reference the font family(ies) specified in the font file.
    Supported font formats are: tff, otf, otc, ttf
  • Databases (currently unused): This Resource type is used for various things in a project. One thing all types have in common is that all of them can be opened in any external text editor to edit the contents.
    Supported database formats are: json, xml
  • Other Formats: This Resource type is used for various things in the project. The flow file is used as storage for the Node Editor's content, the layout file is used for the Player Layouts and the Project data is stored in the JOIPData.json file. One thing they have in common is that all of them can be opened in any external text editor to edit the contents.
    Supported other formats are: json, flow, layout
  • (Archives): This is not an actual resource type but a container containing various resources in a bundle. You can include archived resources in a project to include the contents of said archive in the project as if you had included the separately. Be aware that packing large files in an archive may increase load time when playing the Project.
    Supported archive formats are: zip, 7z, grp, pak, hog, mvl, wad, slb, iso, vdf
    Note: These are the only currently supported archives due to the application requiring fast random access. This list may be extended in future for non-random-access formats.



Online Resources will disable the JOI-Project in the player in Offline Mode and can cause player experience to drop if the internet connection is slow. This type of Resource is not recommended to use.
