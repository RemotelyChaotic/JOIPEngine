JOI-Project

JOI-Projects are a collection of Resource files, a JOIPData.json, a SceneModel.flow file and optionally a Layout file (if a project contains no Layout file, a default Layout file will be used). With the help of script Resources the application can display media Resources according to user defined logic.
A JOI-Project can be played from Play and edited from Editor in the main menu.


JOI-Projects also have a set of properties that can be edited in the Project Properties view of the editor. They are:

  • Project Name: The Project Name is the same as the project folder name and thus must conform to the operating system's folder naming rules and some application specific rules. The project name is not visible outside the editor, so it doesn't have to look nice, just be somewhat describtive.
  • Project Version: This Version is not used for anything internally, but is visible to users in the project selection. It can be used, to keep track of different versions of a released JOI-Project.
  • Target Engine Version: The JOIP-Engine Version this JOI-Project was last saved in. If the engine version doesn't match the curent version, a warning will be shown in the editor. Opening JOI-Projects with a newer engine version in the player or editor might mean, some features are not working correctly, while opening JOI-Projects with an older engine version is not an issue.
  • Describtion: The Describtion can contain html tags for formating and is visible to users in the project selection.
  • Fetishes / Kinks: The set of Kinks / Fetishes, this JOI-Project was tagged with.


JOI-Projects aditionally have a set of hidden properties that are not visible, but influence interactions with the project:

  • Bundled: A bundled project is a project, that was Exported and can only be played or viewed but not edited in the Editor.
