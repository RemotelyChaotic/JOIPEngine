Player Settings

Set various rendering options for the player.

  • Antialiasing: Used by visual elements (like the shape of a message etc.) to decide if the item should use antialiasing or not. In some cases items with antialiasing require more memory and are potentially slower to render.
  • Drop Shadows: Determines whether to use drop shadows for all visual elements in the player or not.
    The soft shadow is created by blurring the elements in a post-processing stop using a gaussian blur. Performing this blur is a costly operation.
  • Mipmaps: This determines whether images use mipmap filtering when scaled or transformed.
    Mipmap filtering gives better visual quality when scaling down compared to smooth, but it may come at a performance cost (both when initializing the image and during rendering).
  • Smoothing: This property holds whether images are smoothly filtered when scaled or transformed. Smooth filtering gives better visual quality, but it may be slower on some hardware. If the image is displayed at its natural size, this property has no visual or performance effect.



Multiple instances of the application share the same configuration file.
If you use different versions of the application, your settings for all instances will change.
