Style

Select the application style. The available styles are in the .\styles subdirectory of the application.



Application needs to be restarted for this to properly take effect.


Multiple instances of the application share the same configuration file.
If you use different versions of the application, your settings for all instances will change.

Styles can be extended with custom styles by sinply adding a folder to the .\styles subrirectory. The style name coresponds to the folder's name. Every style needs a style.css file that needs to be located in the style's root folder (e.g. .\styles\621 Hexagons\style.css). This file can reference resources like images relatively to its own location or absolutely, though absolute paths should not be used.

Additional classes that are useful in a style

  • CMenuButton: The button used in the main menu and primary sub-menus. It's allways drawn directly on the main window background.
  • COverlayBase: Class for overlays, that work like non-modal dialogs and should be styled similarly as a QDialog, as they will be drawn ontop of other widgets.
  • CProjectCardSelectionWidget: The widget displaying the title card of a JOI-Project. Has unique properties:
    • qproperty-selectionColor: The color of the glow of the currently selected card.

  • CTimerWidget: The timer during playing. Note that this is a legacy objetct and not actually in use anymore. Has unique properties:
    • qproperty-primaryColor: Color of the clock segments and the back of the passed time.
    • qproperty-secondaryColor: Color of the clock hand.
    • qproperty-tertiaryColor: Color of the front of the passed time.

  • CScriptEditorWidget: The text editor for scripts. Has unique properties:
    • qproperty-lineNumberBackgroundColor: Background color of the bar with the line numbers.
    • qproperty-lineNumberTextColor: Text color of the line numbers.
    • qproperty-highlightLineColor: Background color of the line currently containing the cursor.
    • qproperty-widgetsBackgroundColor: Background color of the bar with the error icons to the right of the line number bar.
    • qproperty-theme: The theme of the Code Editor. The options are: "Breeze Dark", "Default", "Printing", "Solarized Dark" and "Solarized Light".

  • CEditorSceneNodeWidget: The node editor. Has unique properties:
    • qproperty-normalBoundaryColor: Boundry (border) color of non-selected nodes.
    • qproperty-selectedBoundaryColor: Boundry (border) color of selected nodes.
    • qproperty-gradientColor0: Top color of nodes.
    • qproperty-gradientColor1: Top half backgound color of nodes.
    • qproperty-gradientColor2: Bottom half backgound color of nodes.
    • qproperty-gradientColor3: Bottom color of nodes.
    • qproperty-shadowColor: Shadow color of nodes.
    • qproperty-fontColor: Font color of nodes.
    • qproperty-fontColorFaded: Secondary font color of nodes.
    • qproperty-connectionPointColor: Color of the connection indicator of nodes.
    • qproperty-backgroundColor: Background of the node editor (currently non-functional).
    • qproperty-fineGridColor: Color if the fine grid lines of the node editor.
    • qproperty-coarseGridColor: Color if the coarse grid lines of the node editor.
    • qproperty-normalConnectionColor: Default color of unselected connections.
    • qproperty-selectedConnectionColor: Color of selected connections.
    • qproperty-selectedConnectionHaloColor: Glow color of selected connections.
    • qproperty-hoveredConnectionColor: Color of hovered connections.

Qml Style


Aditionally the qml style of the application (Title Cards and player) can be styled by placing the Control .qml files into the .\qml subfolder inside the style folder (e.g. .\styles\621 Hexagons\qml\). The .\qml subfolder also contains a special configuration file similar to style.css: Style.qml. The Style.qml file is loaded dynamically in qml for style specific configuration.


For general help with qml styling please refer to the qt documentation:

  • https://doc.qt.io/qt-5/qtquickcontrols2-customize.html


Aditionally, there's a set of qml style files, that are required for the application to display properly:

  • Background.png: Background image used by the Player for the default background.
  • TimerBg.svg: Background image used by the Player for the PlayerTimer component.
  • IconBg.svg / IconBgHover.svg: Background images used by the Player for the PlayerIcons, PlayerMetronome and PlayerTextBox components.
  • IconBgMask.png: Image used by the Player for masking Resources in the PlayerIcons component.
  • ButtonExit.svg / ButtonExitHover.svg: Images used by the Player to quit the currently running JOI-Project.
  • ButtonPause.svg / ButtonPauseHover.svg: Images used by the Player to pause a running JOI-Project.
  • ButtonPlay.svg / ButtonPlayHover.svg: Images used by the Player to resume a paused JOI-Project.
  • ButtonArrowDown.svg: Image used by the Title Cards to display arrow to close the detailed view.
  • ButtonArrowUp.svg: Image used by the Title Cards to display arrow to open the detailed view.
  • WarningIcon.svg: Image used by the Title Cards to display that a JOI-Project was created in an incompatible Engine Version.
  • MediaIcon.svg: Image used by the Title Cards to display that a JOI-Project requres video or audio Resources.
  • WebIcon.svg: Image used by the Title Cards to display that a JOI-Project requres web-based Resources.
